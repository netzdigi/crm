# Vista

Прецизно, премиум табло за управление на бизнес операции — фактуриране, клиенти,
приходи по канал и активност на екипа. Изградено с Next.js, React и Tailwind CSS.

## Стек

- Next.js 15 (App Router) + React 19 + TypeScript
- Tailwind CSS за токен-базирана дизайн система (светла/тъмна тема)
- Framer Motion за фини, премерени преходи
- lucide-react за иконите
- Clerk за автентикация (имейл/парола + Google/Facebook/Apple)
- Neon Postgres + Drizzle ORM за мултитенант данните (фирми, членства)

## Разработка

```bash
npm install
npm run dev
```

Отвори [http://localhost:3000](http://localhost:3000).

## Структура

- `app/` — Next.js App Router. `app/(app)/[company]/` са фирмено-обособените страници
  (табло, анализи, клиенти, ...), `app/admin/` е плътформеният панел за собственика на Vista.
- `components/` — компоненти на таблото (странична навигация, горна лента, статистики, графики)
- `lib/data.ts` — примерни данни за таблото (все още не са мултитенант — виж пътната карта в
  плана на проекта)
- `lib/db/` — Drizzle схема, клиент и заявки за фирмите/членствата
- `lib/industries.ts` — кои модули от страничната навигация се активират за всеки бранш

## Настройка на мултитенант средата

1. Добави **Neon Postgres** през Vercel Marketplace към проекта (аналогично на Clerk) —
   Vercel обикновено задава `DATABASE_URL` автоматично.
2. Приложи схемата: `npm run db:push` (или `npm run db:generate` +
   `npm run db:migrate` за версионирани миграции — вече е генерирана първа миграция в
   `lib/db/migrations/`).
3. **Първият плътформен администратор** (собственикът на Vista) трябва да се добави ръчно —
   все още няма UI за това. Влез в Clerk Dashboard → Users, копирай своя `user_id`, после
   изпълни директно в Neon SQL editor:
   ```sql
   INSERT INTO platform_admins (user_id) VALUES ('user_...');
   ```
4. След това `/admin` е достъпен — оттам можеш да създаваш фирми. Полето „Clerk User ID на
   собственика" в „Нова фирма" е по избор — ако го оставиш празно, добави членството по-късно
   по същия начин с директен SQL insert в `memberships`, докато не е готов автоматичният
   flow за покани.
